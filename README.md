# 닷 스틱 (Dot Stick) — 마지막 빛을 가져라

깊은 숲속 빛나는 닷 스틱 10개에서 두 사람이 번갈아 1개 또는 2개씩 가져가고,
**마지막 스틱을 가져가는 사람이 이기는** 가장 단순한 전략 놀이입니다.
(전통 Nim의 뺄셈 변형 — 퍼블릭 도메인, 저작권 문제 없음.)

시각장애 사용자가 화면 없이 **키보드·음성·닷패드 촉각만으로 독립적으로** 즐길 수 있게
설계했습니다. 화면의 핀 미리보기는 닷패드 60×40 출력과 픽셀 단위로 동일합니다.

## 규칙
- 스틱 10개에서 시작
- 자기 차례에 1개 또는 2개를 가져감
- 두 사람이 번갈아 진행
- 마지막 스틱을 가져간 사람이 승리
- 이기는 요령: 상대에게 **3의 배수**를 남기면 됩니다 (선공이면 먼저 1개를 가져가 9개를 남김)

## 조작
| | 키보드 | 닷패드 |
|---|---|---|
| 개수 선택(1↔2) | ← → | 패닝키 좌·우 (또는 F1·F2) |
| 가져가기 | Enter | 기능키 3 |
| 상태 다시 듣기 | F1 | 기능키 4 / 전체 패닝키 |
| 촉각 다시 출력 | F2 | — |
| 도움말 | F3 | — |
| 새 게임 | F4 | — |

화면 왼쪽 아래의 `음성 정지`, `게임 음성 끄기/켜기`, `속도` 버튼으로 자동 안내를 언제든 제어할 수 있습니다. 실제 버튼에 포커스가 있을 때 Enter와 Space는 해당 버튼만 실행합니다.

## 촉각 표현 (60×40)
- 남은 스틱 = 세로 막대. **5개씩 묶어** 세기 쉽게 배치(왼쪽부터 채움)
- 마지막 1개는 더 굵고 길게 — "특별한 빛"으로 구분
- 왼쪽 아래 1·2점 = 선택한 개수, 왼쪽 위 = 탐험가 닷 차례, 오른쪽 위 = 내 차례
- 승리: 빛이 퍼지는 별빛 / 패배: 작은 빈 사각형
- 같은 촉각 그래픽은 중복 전송하지 않으며 F2에서만 강제로 다시 출력
- 모든 상태 변화는 TW_TTS 게임 음성 또는 단일 `aria-live="polite"` 스크린리더 모드로 안내

## 기술 규격 (닷게임즈 준수)
- `index.html` 중심의 정적 구성 (`assets/` 이미지 에셋, 닷패드 SDK 3.0.0 인라인)
- `?embed=1` 임베드 모드 · `postMessage({source:'dotarcade', type:'ready'|'resize'|'exit'})`
- 임베드 모드에서는 음성·결과 조작 패널을 문서 흐름 안에 배치하고 낮은 iframe 높이에서는 내부 스크롤로 겹침 방지
- 음성: 공용 `TW_TTS`(dot-games-host `tts.js`), 실패 시에만 브라우저 음성 폴백
- 지원 해상도: **60×40 (dotpad320)**
- 키보드(방향키·Enter·F1~F4) 전용 조작 · 화면 전환 포커스 관리 · `aria-live` 실시간 안내
- 닷패드(Web Bluetooth)는 **https**에서만 연결됨 (미연결 상태에서도 게임 완전 동작)

## 배포
1. 이 폴더를 본인 GitHub 저장소에 올립니다.
2. Vercel 또는 GitHub Pages로 배포해 https 실행 주소를 만듭니다.
3. 실행 주소 끝에 `?embed=1`을 붙여 음성·키보드 조작을 최종 확인합니다.
